01
What is a PDA?

A Pushdown Automaton extends a finite automaton (FA) with an auxiliary stack. While an FA can only remember its current state, a PDA can push and pop symbols, giving it power to recognise context-free languages (CFLs).

Every regular language is also a CFL, but not vice versa. The canonical example is aⁿbⁿ — a PDA pushes one symbol per 'a' and pops per 'b', accepting when the stack empties.

02
Why It Matters
  • Compilers & Parsers — context-free grammars describe most programming languages; your compiler's parser is a PDA.
  • XML / HTML — matching opening & closing tags is a classic stack problem.
  • Natural Language — sentence structure follows context-free rules.
  • Formal Verification — PDAs underpin many model-checking tools.

06
Acceptance Modes
FINAL STATE

Accept when an accept state is reached after consuming all input. Stack contents irrelevant.

EMPTY STACK

Accept when the stack empties after all input is consumed. State is irrelevant.

Both modes are equivalent in power — any language recognisable by one is recognisable by the other.

07
Stack Operations
↓
PUSH

Pop top, push ≥2 symbols. Stack grows.

↑
POP

Pop top, push nothing. Stack shrinks.

⇄
REPLACE

Pop top, push 1 symbol. Height unchanged.

08
Determinism

A PDA is deterministic (DPDA) if for every configuration (q, a, A) there is at most one applicable transition.

It is nondeterministic (NPDA) if multiple transitions may apply simultaneously.

Unlike finite automata, DPDAs are strictly weaker than NPDAs. Palindromes require nondeterminism. This simulator implements DPDAs only.
